HTML Forms

HTML forms are used to gain some sort of input from the user, and use this information in some way. Examples of form types are input fields, checkboxes, radio buttons, and dropdown menus.

Basic elements of a form:

Tag Name Description
<form> Form The wrapper element for all HTML forms. By default the form will submit to the same HTML page
<input> Input Used to define which type of input is required of the form. Self closing tags
<label> Label Defines a label for many form elements. These are used as input is self-closing, so they give the user some context
Submit We need some way to submit the form, common practice uses buttons

Form tag attributes

Attribute Description Example
action In order for the data in the form to go somewhere other than the page we're already on, the form needs to be actionable. The action attribute links the form to something which will process the data <form action="results.html" >
method Either going to be GET or POST. With a GET request, the entries are appended to the url and sent to another page on your site. POST requests are useful when you have a server, however browsers cannot render POST requests, therefore if you don't have a server, you can only use GET. <form action="results.html" method: GET >
or
<form action="results.html" method: POST >

Input tag attributes

To make an input tag element necessary for the user to complete before the form can be submitted, you can add the required attribute to these tags before closing them! Note: for the input tag we will need to assign all types a name and an id in order for the webpage receiving the action to actually read the data. We will use the id and name "example" in the following examples. We also need the name and id so that we can link these tags to our label tags.

Input type Description Example
<input type="text"> Single line text field <input type="text" name="example" id="example">
<input type="checkbox"> Displays a checkbox. You can have select one or more checkboxes at once. You need a label per checkbox <input type="checkbox" name="example" id="example">
<input type="radio" Displays a radio button. You can only select one radio button at a time. You need a label per radio button. This is done by only having one shared name across the radio elements <input type="radio" name="example" id="example1" value="example1"> <input type="radio" name="example" id="example2" value="example2">
<select></select> While this isn't strictly input, they still allow you to gain a selection from the user, so I'm including this here. It is used as a dropdown selection menu. The individual options are simply
<option value="example">Example</option> . 		<select name="example" id="example"></select>
<input type="submit"> This defines a button for submitting the form data to a form-handler. <input type="submit" value="submit">

Labels

Input type Description Example
<label> Defines many form elements as most have no closing tags. They're also useful for screen-readers. The for attribute should be equal to the id of the relevant input element. <label for="example">
